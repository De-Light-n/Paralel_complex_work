Лабораторна робота No2.

Тема
RESTful сервіси (частина 1). Розробка базового REST API у Spring Boot
Послідовність роботи
1. Ознайомитися з принципами REST-архітектури:
o ресурсна модель;
o URI-дизайн;
o HTTP-методи (GET, POST);
o коди статусів.
2. Обрати предметну область (деталі у “Комплексна робота - теми завдань”).
3. Спроєктувати:
o UML-діаграму класів;
o REST-ендпоінти (таблиця: метод / URL / призначення / код відповіді).
4. Створити веб-проєкт:
o підключити необхідні залежності (Web, Validation тощо);
o налаштувати структуру пакетів (controller, service, model).
5. Реалізувати:
o модель сутностей;
o in-memory зберігання (колекції);
o CRUD-операції (GET all, GET by id, POST).
6. Забезпечити:
o коректні HTTP-коди відповідей;
o валідацію вхідних даних;
o обробку помилок (мінімальний exception handler).
7. Протестувати API за допомогою Postman або curl.
8. Підготувати набір тестових запитів для демонстрації:
o створення ресурсу;
o отримання списку;
o отримання конкретного ресурсу;
o обробка помилки (наприклад, неіснуючий ID).
9. Проаналізувати:
o відповідність REST-принципам;
o структуру JSON-відповідей;
o коректність статус-кодів.
